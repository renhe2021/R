## Bug 修复：Bloomberg BDH 数据问题导致阈值优化全零

### 症状
阈值优化器结果全部为零（Sharpe=0, Return=0%, Alpha=0%），Baseline 和 Best 都是 0。

### 根因分析（通过 3 个递进式调试脚本定位）

**Bug 1 — 日期分裂：** Bloomberg BDH QUARTERLY 把同一季度的不同字段返回在不同日期上。
- Balance Sheet 大多数字段（Total Assets, Equity 等）返回在实际季末日期（如 2025-12-27）
- `EQY_SH_OUT`（股本）返回在月末标准日期（如 2025-12-31）
- `_rows_to_df()` 为每个日期创建独立列 → `_get_latest_col_before()` 选中月末列但只有 shares → ROE=None → 选股失败 → 0 只股票 → 全零

**Bug 2 — 单位不一致：** Bloomberg BDH 返回的**所有**财务报表数值都是百万单位：
- `EQY_SH_OUT = 14,681` → 实际 14,681,000,000 股
- `BS_TOT_ASSET = 379,297` → 实际 $379,297,000,000
- `SALES_REV_TURN = 143,756` → 实际 $143,756,000,000
- `IS_EPS = 2.85` → 实际 $2.85（per-share 字段不需要缩放）
而 yfinance 返回的是实际值（如 Total Assets = 379,297,000,000）

### 修复方案（`src/data_providers/raw_source.py`）

1. **日期合并**：`_rows_to_df()` 将 ±7 天内的日期合并为同一列（以第一个遇到的日期为锚点），确保同一季度的 EQY_SH_OUT 和 BS 字段在同一列
2. **全局单位缩放**：所有 Bloomberg BDH 字段默认 ×1,000,000，除了 per-share/ratio 字段（`IS_EPS`, `PX_LAST`, `EQY_DVD_YLD_IND`, `DVD_SH_LAST`）在 `_BBG_NO_SCALE_FIELDS` 白名单中不缩放
3. **Info dict 修复**：BDP 的 `EQY_SH_OUT` 也同步 ×1,000,000

## 新功能：Stepwise (Coordinate Descent) 搜索方法

### 问题
12 个参数用 Grid Search（4^12 = 16.7M 组合）不现实，Random Search（200 次采样 = 0.001% 覆盖）基本是噪声。

### 解决方案：分组坐标下降 (Stepwise / Coordinate Descent)
将参数按功能模块分成 4 组，逐组做 mini grid search，其他组锁定在当前最优值：

**参数分组 (`PARAM_GROUPS`)**：
1. `screener_gates` (3p) — PE max, D/E max, Market cap min (Stage 3)
2. `school_weights` (最多7p) — Graham, Buffett, Quantitative, Quality, Valuation, Contrarian, GARP 权重 (Stage 5)
3. `forensics` (1p) — High red flags eliminate 阈值 (Stage 4)
4. `scoring_verdict` (4p) — MOS bands, Strong buy/buy score cutoffs (Stage 6-8)

**算法流程**：
- 最多 3 轮外循环 (outer rounds)
- 每轮逐组扫描：每组生成 mini grid (7 steps/param × Cartesian product)，评估每个组合
- 每组扫描完后，锁定该组最优值到 `current_best`
- 收敛检测：Sharpe 改善 < 0.01 时提前终止
- 预算估算：~12×7=84 评估/轮 × 3 轮 ≈ 252 次试验（在 500 次预算内）

**实现文件**：
- `threshold_optimizer.py`:
  - Phase 5 search loop 新增 `stepwise` 分支（与 grid/random 并行存在）
  - `_build_stepwise_groups()` — 将 PARAM_GROUPS 映射到实际 OptimizableParam，含 catch-all 兜底组
  - `_stepwise_mini_grid()` — 为小组参数生成 Cartesian product mini grid
  - `OptimizationConfig.search_method` 默认值改为 `"stepwise"`
- `agent_routes.py`:
  - `OptimizeRequest.searchMethod` 默认值改为 `"stepwise"`，描述更新
  - Walk-Forward 的 `searchMethod` 也支持传 `"stepwise"`（自动透传给内部 ThresholdOptimizer）

### 改进：Stepwise 预算模型（每组上限 → 非全局上限）

**问题**：原来 `max_trials` 是全局上限，4参数组的 mini grid = 4096 组合，500 全局预算在第一个大组就耗尽，后面的组完全搜不到。

**修复**：
- `max_trials` 对 Stepwise 模式改为**每组上限**（如 200/组）
- 如果 mini grid > 每组上限，随机采样截断到上限
- 总消耗估算 = 每组上限 × 组数 × 最大轮数（用于进度条）
- **Bug 修复**：锁定参数时用**组内局部最优**（`grp_best_trial`），而非全局 `best_trial`（全局最优可能来自前一组的参数组合）
- 前端标签改为 "每组试验上限"，默认值 200
- Grid/Random 模式不受影响，仍是全局上限
